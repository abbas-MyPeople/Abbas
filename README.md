# AZ Restaurant Partners — website + knowledge base

The site + strategic brain for **AZ Restaurant Partners** — restaurant-technology consulting by
**Abbas Zoeb**, a software engineer (ex-Google) who also owns an award-winning restaurant
(**Wok & Karahi**, Spring TX). Live at **https://azrestaurantpartners.com** (GitHub Pages, static,
no build step). *(Older internal name "AZ Integrations" survives in some research headers — the live
brand is AZ Restaurant Partners.)*

Positioning: *a restaurant family plus the engineering team you could never hire — we set up only the
few tools that pay off, transparently and done-for-you, and you only pay once you're saving.*

## The two-business model (read this first)
Wok & Karahi is Abbas's **test business**. Capabilities are built and proven there, then productized
and sold here. The two live in **separate repos on purpose**:
- **Restaurant (the lab):** `abbas-MyPeople/WokAndKarahiTexas.com`
- **AZ (the product):** `abbas-MyPeople/Abbas` (this repo)

The proof that flows between them lives in **[`flagship/`](flagship/)** — start at
[`flagship/CAPABILITY-LEDGER.md`](flagship/CAPABILITY-LEDGER.md) (what we've proven → what we sell →
the gaps) and [`flagship/SYNC-PROTOCOL.md`](flagship/SYNC-PROTOCOL.md) (how the bridge stays current).

## Working here
- **No build step.** Edit → commit → push `main` → GitHub Pages publishes.
- **Regenerate the tool directory** after changing tool data: `node scripts/gen-toolbox.js` (rewrites
  the directory + per-tool pages + tool URLs in the sitemap between markers).
- **Ground every claim in `flagship/CAPABILITY-LEDGER.md`** — don't market a capability the flagship
  hasn't proven (or label it honestly as advisory). See the ledger's status legend.
